## Step 0: Determine Week

```bash
NEXT_WEEK=$(date -v+1d +%Y-W%V)  # On Sunday, +1d = Monday of next week
```

## Step 1: Collect Data

### 1a. Next Week Calendar (5 business days)
Read calendar events for the next 7 days. Filter to Monday through Friday. Identify:
- Recurring meetings vs. one-offs
- Back-to-back blocks
- Heavy vs. light days
- Any conflicts

### 1b. Email Backlog
Count total unread. Flag any >3 days old as stale risk.

### 1c. Open Tasks + Retrospective Carry-Forward

Read recent daily notes (last 3 days) for unchecked tasks carrying over.

Also check if the weekly retrospective has already run:
```bash
THIS_WEEK=$(date +%Y-W%V)
```
If `vault/daily/$THIS_WEEK/$THIS_WEEK.md` exists and contains a `### Carry-Forward` section, read it and incorporate those items into Open Tasks and Key Meetings to Prep.

### 1d. Slack Carryover
Read `state/slack-digest.json` for any unresolved blocking threads.

### 1e. GitHub Projects (if configured)
Check tracked repos for activity and momentum.

## Step 2: Archive This Week's Daily Notes

Move daily notes from vault root into a weekly subfolder:

```bash
THIS_WEEK=$(date +%Y-W%V)
mkdir -p vault/daily/$THIS_WEEK/

cd vault
for f in 20[0-9][0-9]-[0-9][0-9]-[0-9][0-9].md; do
  [ -f "$f" ] && mv "$f" "daily/$THIS_WEEK/"
done
```

Report: "Archived N daily notes to vault/daily/{THIS_WEEK}/"
